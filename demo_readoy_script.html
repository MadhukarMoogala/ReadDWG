üèóÔ∏è Project Overview
The ReadDWG pipeline is designed to automate DWG file processing for web-based applications and backend services. Whether it's a simple layer extraction, block analysis, or integration into a larger BIM workflow — this system does it in a clean, asynchronous fashion.

At its core is a lightweight .NET Web API endpoint. It accepts DWG file uploads, spins off a background job to parse and process the drawing, and then exposes endpoints to:

Check the job status

Retrieve structured results like layers, blocks, and geometry metadata

This is backed by a console app that uses Autodesk’s RealDWG SDK to perform actual DWG parsing. The console app runs in a separate process, ensuring safe, isolated execution — perfect for multi-threaded or microservice setups.

This setup is modular and decoupled — meaning it can be dropped into an existing pipeline or scaled independently with containers or job runners.

üìä Load Testing with K6 (Demo)
Now, let’s talk about how we validate performance and resilience.

We use K6, an open-source load testing tool that allows us to simulate real-world traffic patterns.

In this demo, we’ve written a K6 script that simulates 10 virtual users concurrently uploading DWG files, polling for job completion, and retrieving results — just like a production app would.

Each user randomly picks from a set of DWG samples. The script checks:

That each upload returns a valid job ID,

That the job status transitions correctly,

And that the final result contains both layers and blocks.

We run the test for 1 minute, and we define thresholds to ensure quality:

95% of requests must complete within 5 seconds,

and 95%+ of all checks must pass.

At the end, we automatically generate a beautiful HTML report using k6-reporter, which gives us visual breakdowns of response times, pass/fail counts, and system load over time.
